## 事例1

| 項目 | 内容 |
|------|------|
| Before画像 | IMG_6502.JPG |
| After画像 | IMG_6503.JPG |
| 使用シャンプー | パープル 単品 |
| 使用トリートメント | ラベンダー + パープル = 1:1 |
| 放置時間 | 5分 |
| 施術履歴 | （記載なし） |

### 観察
- Before: 明るめの茶髪（推定明度10〜12レベル程度）
- After: 黄みが抑えられ、落ち着いたトーンに変化
- シャンプーとトリートメントで異なるカラーを使用している
- トリートメントは2色ミックス（配合比率 1:1）

## 事例2

| 項目 | 内容 |
|------|------|
| Before画像 | IMG_6504.JPG |
| After画像 | IMG_6505.JPG |
| 使用シャンプー | （記載なし） |
| 使用トリートメント | ブラウン 単品 |
| 放置時間 | 5分 |
| 施術履歴 | 縮毛矯正（アンダー部分）、毎日アイロン使用 |

### 観察
- Before: 暗めの茶髪、やや赤みあり
- After: ツヤが出て整った印象、ブラウンが補色された
- 施術履歴として「縮毛矯正」「毎日アイロン」が記載 → 髪のダメージ状態の参考情報
- トリートメントのみの使用（シャンプーの記載なし）

## サンプルから見えるデータ構造の特徴

1. **シャンプーとトリートメントは別々に記録する必要がある**
   - 片方だけ使用するケースがある
   - それぞれ別のカラーを使うケースがある

2. **ミックス（配合）の記録が必要**
   - 単品使用と複数色ミックスの両方がある
   - ミックス時は色名と配合比率を記録

3. **施術履歴は自由度が高い**
   - 「縮毛矯正」「毎日アイロン」など、定型＋自由記述が混在
   - 髪のどの部分かの情報もある（「アンダー」=下部分）

4. **画像内にテキスト情報が記載されている**
   - 現状は写真にテキストを載せて共有している運用
   - アプリ化により、テキスト情報を構造化データとして管理できるようになる
