# irop 施術事例集 — PRD（プロダクト要件定義書）

## 1. プロダクト概要

### 1.1 プロダクト名
irop 施術事例集（仮称）

### 1.2 背景と課題
irop（イロップ）はカラーシャンプー・カラートリートメントを美容室に提供するヘアケアブランド。美容室への営業活動において、美容師がiropの導入を検討する際の最大の障壁は：

> **「どんな色相・明度の髪に、どの商品を使うと、どんな仕上がりになるのか」がわからない**

現状は施術事例の写真にテキストを重ねて共有する運用だが、事例が増えると検索性・管理性が低下する。

### 1.3 解決策
施術事例をWebアプリ上で一覧化し、Before/Afterの写真と施術情報を構造化データとして管理する。美容師が自分の顧客の髪の状態に近い事例をフィルター検索で素早く見つけられるようにする。

### 1.4 目標指標
- 美容師への営業時に「事例集」として活用でき、導入検討の後押しになる
- 10件の事例からスタートし、継続的に事例を蓄積できる運用体制を構築する

---

## 2. ユーザーと役割

| 役割 | 誰か | できること |
|------|------|----------|
| 閲覧者 | 美容師（誰でも） | 事例の一覧閲覧、フィルター検索、事例詳細の閲覧 |
| 投稿者 | 招待された美容師 | 上記 + 施術事例の新規登録 |
| 管理者 | iropブランドマネージャー | 上記 + 事例の承認/却下、美容師の招待、事例の編集/削除 |

### アクセスポリシー
- **閲覧**: URLを知っていれば誰でも可能（ログイン不要）
- **登録**: 招待制。管理者が招待した美容師のみ
- **承認フロー**: 投稿者が登録した事例 → 管理者が確認・承認 → 公開

## 3. 機能要件

### 3.1 施術事例の閲覧（閲覧者向け）

**事例一覧画面**
- 施術事例をカード形式で一覧表示
- 各カードにBefore/After写真のサムネイル、主要情報（明度、使用カラーなど）を表示
- フィルター機能で絞り込み可能

**事例詳細画面**
- Before/After写真を大きく並べて比較表示
- 施術情報の全項目を表示

### 3.2 フィルター検索（閲覧者向け）

| フィルター項目 | 検索方式 |
|-------------|---------|
| 髪の明度 | 範囲指定（例: 5〜7レベル） |
| 髪の色相 | 選択式（16色相グループ、複数選択可） |
| 白髪率 | 選択式（10%未満 / 10-20% / 30-50% / 50-70% / 70%以上） |
| 施術履歴 | チェックボックス（複数選択可） |
| 使用したiropカラー | チェックボックス（5色から選択） |

### 3.3 施術事例の登録（投稿者・管理者向け）

スマートフォンから施術直後、または後からまとめて登録できるUI。

#### 登録データ項目

**Before（施術前）**

| 項目 | 入力形式 | 必須 | 詳細 |
|------|---------|------|------|
| Before写真 | 画像アップロード | − | 施術前の髪の写真（任意。未登録時はグレー背景「No Image」表示） |
| 髪の明度 | 数値入力（レベル） | ○ | 1〜20程度の数値 |
| 髪の色相 | 選択式（16種）＋自由記述 | ○ | PCCS準拠の16色相グループから選択 |
| 白髪率 | 選択式 | ○ | 10%未満 / 10-20% / 30-50% / 50-70% / 70%以上 |
| 施術履歴 | チェックボックス＋自由記述 | − | 縮毛矯正 / パーマ / ブリーチ / 通常カラー / その他 |
| 備考 | 自由記述 | − | 補足情報（例:「毎日アイロン使用」「アンダー部分のみ縮毛矯正」） |

**After（施術後）**

| 項目 | 入力形式 | 必須 | 詳細 |
|------|---------|------|------|
| After写真 | 画像アップロード | ○ | 施術後の髪の写真 |
| 使用シャンプー | カラー選択＋ミックス対応 | − | PINK/LAVENDER/ASH/BROWN/PURPLEから選択。ミックス時は色＋配合比率。未使用も可 |
| 使用トリートメント | カラー選択＋ミックス対応 | − | 同上 |
| 放置時間 | 数値入力（分） | ○ | 例: 5分、10分 |

### 3.4 管理機能（管理者向け）

- **承認管理**: 投稿された事例の一覧表示、承認/却下
- **事例編集・削除**: 公開済み事例の編集、非公開化、削除
- **招待管理**: 美容師への招待リンク発行

## 4. 髪の色相分類（全16グループ）

irop診断ロジック（PCCS準拠）に基づく分類。登録時の選択肢およびフィルター検索に使用。

**色相環ベース（11種）**
イエロー系 / 金髪・ブロンド系 / カーキ系 / グリーン系 / アッシュ系 / ブルー系 / パープル系 / ピンク系 / レッド系 / オレンジ系 / ベージュ系

**無彩色・特殊系（5種）**
ホワイト系 / シルバー系 / グレージュ系 / ブラック系 / ブラウン系

## 6. 非機能要件

### 6.1 プラットフォーム
- Webアプリケーション
- **モバイルファースト**（美容師がスマートフォンで閲覧・登録）

### 6.2 ドメイン・ホスティング
- irop.jp の一部として公開（例: salon.irop.jp）
- 既存サイト: irop.jp は ecforce で運営
- ドメイン管理: Xserver
- → サブドメインを Xserver で設定し、アプリのホスティング先に向ける構成

### 6.3 デザインガイドライン
- iropブランドガイドライン（CIVI Guideline）に準拠
- **フォント**: Typo Round（英数字指定、埋め込み不可時はComfortaa代用）、Zen Kaku Gothic New（和文、および日本語混在・可読性重視時の英字）
- **トーン**: シンプルで上質、クリーン
- **キーモチーフ**: カラーバー（先端が正円の棒状モチーフ、5色で展開）
- **写真**: 上質でシンプル、髪色がきれいに見えるトーン

### 6.4 規模感
- 初期: 10件程度
- 将来: 数百件規模を想定
