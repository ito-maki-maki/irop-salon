# irop 施術事例集 アプリケーション概要

## 目的

美容師がiropの導入を検討する際に、「どんな髪に、どの商品を使うと、どんな仕上がりになるのか」を具体的にイメージできるようにする。施術事例をWeb上で一覧・フィルター検索できるアプリケーション。

## 想定ユーザー

- **閲覧者**: 美容師（スマートフォンでの閲覧が主）
- **登録者**: iropブランドマネージャー + 美容師本人（どちらも事例を登録できる）

## 主要機能

### 1. 施術事例の閲覧（美容師向け）

各事例で以下のBefore/Afterを比較できる:

#### Before（施術前）
- 施術前の髪の写真
- 髪の状態情報
  - 明度（レベル）
  - 白髪の有無・白髪率
  - 施術履歴（縮毛矯正、パーマ、ブリーチなど）
  - その他、美容師が参考にしたい基礎情報

#### After（施術後）
- 施術後の髪の写真
- 使用したiropアイテム（シャンプーとトリートメントを別々に記録）
  - **シャンプー**: カラー名（単品 or ミックス+配合比率）※使用しない場合もある
  - **トリートメント**: カラー名（単品 or ミックス+配合比率）※使用しない場合もある
- 放置時間

#### データ入力について
- Before情報（明度、白髪率など）は手動入力
- 画像はアップロード

### 2. フィルター検索（美容師向け）

施術事例一覧を、以下の項目で絞り込みできる:
- 髪の明度
- 髪の色
- 施術履歴（縮毛矯正、パーマなど）
- 白髪の有無・白髪率
- 使用したiropのカラー

### 3. 事例データの登録・更新（ブランドマネージャー＋美容師）

- 施術事例が増えるたびに、画像や情報をアップロードできる
- ブランドマネージャーと美容師のどちらも登録可能
- （詳細な管理画面・権限の仕様は今後検討）

## プラットフォーム

- Webアプリケーション
- スマートフォンでの閲覧を最優先（モバイルファースト）

## サンプル事例から判明したデータ構造の特徴

実際のサンプル画像（`images/`フォルダ）から以下が判明:

1. シャンプーとトリートメントで**別々のカラーを使う**ケースがある（例: シャンプー=パープル、トリートメント=ラベンダー+パープル）
2. **片方のみ使用**するケースがある（例: トリートメントのみ使用）
3. **ミックス配合**がある（例: ラベンダー+パープル 1:1）
4. 施術履歴は**定型項目＋自由記述**が混在（例:「縮毛矯正」「毎日アイロン」）
5. 髪の**部位情報**がある場合がある（例:「アンダー」=下部分に縮毛矯正）
